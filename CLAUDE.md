## פרוטוקול תיקון קוד → פרודקשן (חובה!)

כל תיקון קוד חייב להסתיים ברצף סגור. **אם אחד מהשלבים לא בוצע — אסור לסיים את התשובה.**

### השלבים:

1. **Build מקומי מוצלח** — `npm run build` או `npx vite build` ללא שגיאות.
2. **Commit + Push** — `git add` → `git commit -m "..."` → `git push origin main`.
3. **אימות הגעה לפרודקשן** — להריץ `git log --oneline -3` אחרי ה-push ולוודא שהקומיט החדש שם, ולהזכיר למשתמש לבדוק שה-build ב-Vercel הצליח.
4. **הוראות לטעינה מחדש** — להזכיר למשתמש hard-refresh (Cmd+Shift+R), ואם זה PWA: DevTools → Application → Service Workers → Unregister.

### מה אסור לעשות:

- ❌ אסור לכתוב "הרץ את הפקודות האלה" בלי לאמת בסוף שהן באמת רצו.
- ❌ אסור להניח שהמשתמש הריץ commit/push בלי לבדוק `git status` או `git log`.
- ❌ אסור לסיים תיקון רב-קבצי בלי TodoList שמסמן כל קובץ + כל שלב.

### מה כן לעשות:

- ✅ אם המשתמש לא נתן אישור מפורש לדחיפה, **לבקש אישור פעם אחת קצרה** ("לרוץ commit+push? כן/לא") ואז להריץ בעצמי.
- ✅ לפני סיום: `git status` + `git log -1` כדי לוודא שאין שינויים תלויים באוויר.
- ✅ אם זה תיקון UI שעובר דרך Service Worker — להזהיר במפורש על cache.

## בקשות רב-קבציות / רב-ממשקיות

כשהבקשה מזכירה "כל הממשקים", "שלושת הממשקים" (מתאמן/מאמן/מנהל), או "בכל המקומות":
- **ליצור TodoList עם משימה נפרדת לכל ממשק/קובץ** — לא לעבד הכל כמשימה אחת.
- **לסיים בטבלה מסכמת**: קובץ + שורה + מה שונה, לכל הקבצים שנגעו.
- **אם הצלחתי בחלק ולא בכולם — לאמר את זה במפורש** ולא להמשיך הלאה.

## SQL / מיגרציות — חובה!

**כל פעם שצריך להריץ SQL ב-Supabase SQL Editor** — לכתוב את כל ה-SQL **ישירות בתוך התשובה בבלוק קוד להעתקה** (```sql ... ```), עם הוראה ברורה "להעתיק ולהדביק ב-SQL Editor".

❌ **אסור לחלוטין** לשלוח רק את הנתיב לקובץ (`/Users/.../migration.sql`) ולצפות מהמשתמש להעתיק ממנו — Supabase SQL Editor מקבל SQL בלבד, לא נתיבים.

✅ גם אם הקובץ נשמר בריפו (לתיעוד), עדיין **חובה** להציג את התוכן המלא בתשובה כקופי-פייסט מוכן.

## קונטקסט עסקי

- אפליקציית TeamPact — מערכת לאקדמיית BJJ של דודי בן זקן (Team Pact).
- שלושה ממשקים: **מתאמן** (`AthleteDashboard`), **מאמן** (`TrainerDashboard`), **מנהל** (אותו `TrainerDashboard` עם `isAdmin=true`).
- Stack: React + Vite + Supabase + Tailwind. PWA עם Service Worker.
- Deploy: GitHub `main` → Vercel auto-deploy.
- VAT ישראל: 18%.
